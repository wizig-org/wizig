# {{APP_NAME}}

Generated by `wizig create`.

## Project Layout

- `lib/`: Zig app business/domain logic.
- `ios/`: iOS host app (SwiftUI + Wizig SDK package).
- `android/`: Android host app (Compose + generated bindings).
- `.wizig/sdk/`: vendored Wizig host SDK wrappers.
- `.wizig/runtime/`: vendored Zig runtime/FFI sources.
- `.wizig/generated/`: generated bridge artifacts (JNI, Zig, and mirrors).
- `wizig.api.zig` / `wizig.api.json` (optional): explicit contract overrides.
- `plugins/`: local plugin packages and manifests.

## Typical Workflow

1. Add or update `pub fn` APIs in `lib/**/*.zig` (for example `lib/main.zig`).
2. Regenerate bindings with `wizig codegen` (or run `wizig codegen --watch` during IDE editing).
3. Run app with `wizig run`.

## Commands

```sh
wizig codegen .
wizig codegen . --watch
wizig plugin sync .
wizig run . --once
```

## Notes

- Generated files under `.wizig/generated/` are build artifacts; do not hand-edit.
- Generated SDK mirrors under `.wizig/sdk/**/WizigGeneratedApi.*` are build artifacts.
- If iOS project sources look stale, run `xcodegen generate` in `ios/`.
