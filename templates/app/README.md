# {{APP_NAME}}

Generated by `wizig create`.

## Project Layout

- `lib/`: Zig app business/domain logic.
- `ios/`: iOS host app (SwiftUI + Wizig SDK package).
- `android/`: Android host app (Compose + generated bindings).
- `.wizig/sdk/`: vendored Wizig host SDK wrappers.
- `.wizig/runtime/`: vendored Zig runtime/FFI sources.
- `.wizig/generated/`: generated typed bridge and plugin registrants.
- `wizig.api.zig`: canonical API contract for codegen.
- `plugins/`: local plugin packages and manifests.

## Typical Workflow

1. Edit contract in `wizig.api.zig`.
2. Regenerate bindings with `wizig codegen`.
3. Run app with `wizig run`.

## Commands

```sh
wizig codegen .
wizig plugin sync .
wizig run . --once
```

## Notes

- Generated files under `.wizig/generated/` are build artifacts; do not hand-edit.
- If iOS project sources look stale, run `xcodegen generate` in `ios/`.
