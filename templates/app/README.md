# {{APP_NAME}}

Generated by `ziggy create`.

## Project Layout

- `lib/`: Zig app business/domain logic.
- `ios/`: iOS host app (SwiftUI + Ziggy SDK package).
- `android/`: Android host app (Compose + generated bindings).
- `.ziggy/sdk/`: vendored Ziggy host SDK wrappers.
- `.ziggy/runtime/`: vendored Zig runtime/FFI sources.
- `.ziggy/generated/`: generated typed bridge and plugin registrants.
- `ziggy.api.zig`: canonical API contract for codegen.
- `plugins/`: local plugin packages and manifests.

## Typical Workflow

1. Edit contract in `ziggy.api.zig`.
2. Regenerate bindings with `ziggy codegen`.
3. Run app with `ziggy run`.

## Commands

```sh
ziggy codegen .
ziggy plugin sync .
ziggy run . --once
```

## Notes

- Generated files under `.ziggy/generated/` are build artifacts; do not hand-edit.
- If iOS project sources look stale, run `xcodegen generate` in `ios/`.
